01. 비동기

        자바스크립트는 한 번에 하나의 태스크(작업 단위)만 실행할 수 있는
        싱글 쓰레드 방식으로 동작한다.
        그렇기 때문에 처리에 시간이 걸리는 태스크를 실행하는 경우
        블로킹(작업 중단)이 발생하게 된다.

동기 처리(synchronous)

        현재 실행 중인 태스크(작업) 종료할 때까지 다음에 실행될
        태스크가 대기하는 방식을 동기 처리라고 한다.
        장점 : 태스크(작업) 순서대로 하나씩 처리하기 때문에 실행 순서가 보장됨.
        단점 : 앞선 태스크가 종료할 때까지 이후 것들이 블로킹 된다.

비동기 처리(asynchronous)

        현재 실행 중인 태스크가 종료되지 않은 상태라고 해도
        다음 태스크를 곧바로 실행하는 방식을 비동기 처리라고 한다.
        장점 : 태스크가 블로킹 되지 않는다.
        단점 : 태스트의 실행 순서가 보장되지 않는다.

        setTimeout, setInterval, 이벤트 핸들러, HTTP 요청
        등은 비동기 처리 방식으로 동작한다.
